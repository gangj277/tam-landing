# 딥다이브 인사이트 활용 계획

> 작성일: 2026-04-02
> 상태: 설계 완료, 구현 대기

---

## 현재 상태

딥다이브 에이전트가 대화 중 아이의 유의미한 발언을 `save_insight` tool로 기록함.
`deep_dive_insights` 테이블에 영구 저장됨.

### 저장되는 데이터

```typescript
{
  id: "ddi-xxxxx",
  deepDiveId: "dd-xxxxx",
  text: "자원 배분에서 효율보다 인간의 긴급한 필요를 우선시하는 가치관",  // 사고의 본질 요약
  sourceMessageIndex: 5,     // 어떤 대화 턴에서 나온 건지
  valueTags: ["fairness", "empathy"],  // 가치 태그
  createdAt: "2026-04-02T..."
}
```

### save_insight 기준 (프롬프트에 명시됨)
- ✅ 자기만의 가치관/판단 기준을 드러냈을 때
- ✅ 대화 중 관점이 바뀌거나 깊어졌을 때
- ✅ 복잡한 상황의 본질을 스스로 포착했을 때
- ✅ 자기 성향/패턴을 인식하는 발언을 했을 때
- ❌ 단순 동의, 반복, 표면적 대답은 저장 안 함

## 활용 루프 3개

### 루프 1: 다음 딥다이브 대화에서 참조

**목적:** 에이전트가 이전 인사이트를 인지하고, 아이의 성장 맥락을 이어감

**구현 방법:**
- `buildAgentSystemPrompt()`에 이전 인사이트 섹션 추가
- `streamAgentResponse()`에서 해당 아이의 최근 인사이트 5-10개를 로드
- 시스템 프롬프트에 주입:

```
<이전 딥다이브에서의 발견>
이 아이는 지금까지의 딥다이브에서 다음과 같은 생각을 보여줬어:
- "자원 배분에서 효율보다 인간의 긴급한 필요를 우선시" (fairness, empathy)
- "리더는 명령하는 사람이 아니라 듣는 사람" (leadership, empathy)
- "정답이 없는 문제에서도 기준은 있어야 한다" (logic, independence)

이 맥락을 기억하되, 강제로 언급하지는 마.
아이의 오늘 대화와 자연스럽게 연결될 때만 참고해.
예: "지난번에 '공평함'에 대해 깊이 생각했잖아. 오늘 이 상황에서도 비슷한 고민이 있을까?"
</이전 딥다이브에서의 발견>
```

**수정할 파일:**
- `lib/server/ai/prompts.ts` — `buildAgentSystemPrompt` 파라미터에 `previousInsights` 추가
- `lib/server/services/deep-dive.ts` — `streamAgentResponse`에서 `store.listDeepDiveInsights` 호출 (다른 딥다이브의 인사이트)

**우선순위:** 높음 — 대화 깊이에 직접 영향

---

### 루프 2: 미션 생성/추천에 반영

**목적:** 아이의 관심 패턴에 맞는 미션을 추천/생성

**구현 방법:**
- 인사이트의 `valueTags` 빈도를 집계
- 미션 추천 시 가중치로 사용

```typescript
// 인사이트에서 valueTags 빈도 집계
const tagCounts = insights.reduce((acc, i) => {
  i.valueTags.forEach(tag => acc[tag] = (acc[tag] || 0) + 1);
  return acc;
}, {});

// 결과 예시:
// { fairness: 5, empathy: 4, logic: 3, creativity: 2, ... }
```

**활용 전략:**
| 전략 | 설명 |
|------|------|
| **강화** | 높은 태그 관련 미션 → 몰입도 높일 때 |
| **확장** | 낮은 태그 관련 미션 → 새로운 영역 노출할 때 |
| **도전** | 상충하는 태그 조합 미션 → 복잡한 사고 유도할 때 |

예시:
- fairness + empathy가 높은 아이 → "효율 vs 공정" 딜레마 미션 (도전)
- creativity가 낮은 아이 → 창작/표현 미션 (확장)
- adventure가 높은 아이 → 모험형 미션 (강화, 몰입 유지)

**수정할 파일:**
- `lib/server/missions/assignment.ts` — 미션 추천 로직에 인사이트 태그 반영
- `lib/server/services/deep-dive.ts` — 아이별 인사이트 태그 집계 함수 추가

**우선순위:** 중간 — 14일 온보딩 이후 개인화에 필요

### 루프 3: 내 기록 탭에서 열람

**목적:** 아이가 자기 성장을 스스로 돌아볼 수 있음

**UI 설계:**

```
내 기록 탭
├── 미션 기록 (기존)
├── 딥다이브 기록 (신규)
│   ├── 3월 4주 — 민재와의 대화 (화성 도시)
│   │   ├── 포트폴리오: "자원을 나눌 때는 모두의 의견을 들어야 한다"
│   │   └── 발견된 것들:
│   │       · "효율보다 인간의 긴급한 필요를 우선시"
│   │       · "결정 과정에서 합의가 결과보다 중요"
│   ├── 4월 1주 — 수진과의 대화 (동물 구조센터)
│   │   ├── 포트폴리오: "..."
│   │   └── 발견된 것들: ...
│   └── ...
└── 관심 지도 (기존, 인사이트 태그 반영 강화)
```

**데이터 소스:**
```typescript
// API: GET /api/portfolio?childId=xxx
// 현재: portfolioEntry만 반환
// 변경: portfolioEntry + insights 포함

{
  entries: [
    {
      deepDiveId: "dd-xxx",
      missionTitle: "화성 첫 도시의 시장",
      expertName: "민재",
      portfolioEntry: "자원을 나눌 때는 모두의 의견을 들어야 한다",
      insights: [
        { text: "효율보다 인간의 긴급한 필요를 우선시", valueTags: ["fairness", "empathy"] },
        { text: "결정 과정에서 합의가 결과보다 중요", valueTags: ["community", "logic"] },
      ],
      completedAt: "2026-03-31T..."
    }
  ]
}
```

**수정할 파일:**
- `lib/server/services/deep-dive.ts` — `getPortfolioByChild`에 insights 포함
- `app/(app)/profile/page.tsx` — 포트폴리오 섹션에 인사이트 카드 추가
- `lib/api-client.ts` — PortfolioEntry 타입에 insights 추가

**우선순위:** 높음 — 아이의 자기이해 경험에 직결

## 추가 활용 (향후)

### 부모 주간 리포트
```
이번 주 딥다이브에서 발견된 것:
· 서연이는 '공평함'에 대해 깊이 고민했어요.
  "효율보다 사람의 필요가 먼저"라는 가치관을 보여줬습니다.
· 대화 중 관점이 바뀌는 순간이 있었어요.
  처음엔 "똑같이 나누면 된다"고 했다가,
  "상황에 따라 달라질 수 있다"로 생각이 넓어졌습니다.
```

### 고교학점제 과목 추천 강화
인사이트 태그 패턴 → 과목 매핑:
- fairness + community → 사회문제탐구, 법과정치
- creativity + independence → 미술 전공 실기, 영상 제작
- logic + efficiency → 수학과제 탐구, 데이터 과학

## 핵심 원칙

1. **인사이트는 아이의 것이다** — 평가가 아니라 발견. "잘했다/못했다"가 아니라 "이런 생각을 했다"
2. **강제로 만들지 않는다** — 유의미한 순간이 없으면 없는 대로. 숫자 채우기 금지
3. **맥락과 함께 보여준다** — 인사이트만 덜렁 보여주면 의미가 없음. 어떤 미션에서, 어떤 대화 맥락에서 나온 건지 함께 표시
4. **시간순으로 성장이 보여야 한다** — 3월의 인사이트와 6월의 인사이트를 비교하면 아이의 사고가 어떻게 변했는지 보임
