# 탐 (TAM) 앱 기획서 v0.1

> 이 문서는 더미 데이터 기반 프론트엔드 인터페이스를 설계하기 위한 기획서입니다.
> 실제 개발 착수 전, 데이터 구조와 사용자 흐름을 확정하는 것이 목적입니다.

---

## 1. 핵심 전제

### 타겟 사용자
- **주 사용자**: 초등 4학년 ~ 중학 2학년 (10~14세)
- **결제자/관찰자**: 부모
- **디바이스**: 태블릿 > 모바일 > 데스크탑 (모바일 퍼스트로 설계)

### 제품 핵심 루프
```
매일 1개 미션 (5~10분)
→ 세계 진입 (Encounter)
→ 상황 이해 + 선택 (Choice)
→ AI와 함께 확장/행동 (Act)
→ 성찰 피드백 (Mirror)
```

### 설계 원칙
1. **정답 없음** — 모든 선택은 가치의 표현이지 점수가 아니다
2. **평가 아닌 거울** — AI는 아이의 패턴을 비추어 줄 뿐 판단하지 않는다
3. **10분 안에 끝나야 한다** — 긴 세션은 이탈을 부른다
4. **아이 말투** — UI 텍스트는 존댓말 없이, 친구처럼, 하지만 유치하지 않게

## 3. 데이터 모델

### 3-1. Mission (미션)

```typescript
interface Mission {
  id: string;                    // "mission-mars-mayor"
  title: string;                 // "화성 첫 도시의 시장"
  role: string;                  // "도시 리더"
  category: MissionCategory;
  difficulty: DifficultyType;

  // 세계관 설정
  worldSetting: {
    location: string;            // "화성 도시 아레스"
    era: string;                 // "2147년"
    backdrop: string;            // 분위기 묘사 1~2줄
  };

  // 상황 + 핵심 질문
  situation: string;             // 3~4줄의 상황 설명
  coreQuestion: string;          // 아이에게 던지는 핵심 질문

  // 선택지 (2~3개, 정답 없음)
  choices: Choice[];

  // AI 대화 시스템 프롬프트 컨텍스트
  aiContext: {
    persona: string;             // AI의 역할 ("이 세계의 부시장으로서 조언을 구하러 왔어")
    followUpAngles: string[];    // AI가 파고들 수 있는 방향들
    expansionTools: ExpansionTool[];  // 더 넓혀보기 / 다른 시각 / 이상하게 바꾸기
  };

  // 메타
  tags: string[];                // ["우주", "리더십", "자원관리"]
  estimatedMinutes: number;      // 5~10
  ageRange: [number, number];    // [10, 14]
}

type MissionCategory =
  | "world"        // 세계 탐험
  | "value"        // 가치 선택
  | "perspective"  // 관점 전환
  | "real"         // 현실 연결
  | "synthesis";   // 표현

type DifficultyType =
  | "value-conflict"      // 가치 충돌형
  | "dilemma"             // 딜레마형
  | "creative-judgment"   // 창작 판단형
  | "perspective-shift"   // 관점 전환형
  | "emotional-immersion" // 감정 몰입형
  | "observation"         // 관찰 분석형
  | "problem-discovery"   // 문제 발견형
  | "expression"          // 표현 도전형
  | "comprehensive";      // 종합 창작형
```

### 3-2. Choice (선택지)

```typescript
interface Choice {
  id: string;
  label: string;            // "농장에 먼저 물을 보낸다"
  shortLabel: string;       // "농장 우선" (카드 UI용)
  reasoning: string;        // 이 선택의 논리 힌트 (선택 후 보여줌)
  valueTags: ValueTag[];    // 이 선택이 반영하는 가치들
}

type ValueTag =
  | "fairness"       // 공정성
  | "efficiency"     // 효율
  | "safety"         // 안전
  | "adventure"      // 모험
  | "empathy"        // 공감
  | "creativity"     // 창의성
  | "independence"   // 독립성
  | "community"      // 공동체
  | "logic"          // 논리
  | "emotion";       // 감정
```

### 3-3. ExpansionTool (AI 확장 도구)

```typescript
interface ExpansionTool {
  type: "broaden" | "reframe" | "subvert";
  label: string;           // "더 넓혀보기" | "다른 시각으로 보기" | "이상하게 바꾸기"
  icon: string;            // 이모지 or 아이콘 이름
  prompts: string[];       // AI에게 보낼 프롬프트 변형들
}

// 예시:
// { type: "broaden",  label: "더 넓혀보기",      icon: "🔭" }
// { type: "reframe",  label: "다른 시각으로 보기", icon: "🔄" }
// { type: "subvert",  label: "이상하게 바꾸기",   icon: "🌀" }
```

### 3-4. MissionSession (체험 기록)

```typescript
interface MissionSession {
  id: string;
  missionId: string;
  userId: string;
  startedAt: string;       // ISO timestamp
  completedAt?: string;

  // 선택 기록
  choicesMade: {
    choiceId: string;
    timestamp: string;
    reflectionNote?: string;   // 아이가 직접 쓴 이유 (선택)
  }[];

  // AI 대화 기록
  conversation: ConversationMessage[];

  // 사용한 확장 도구
  toolsUsed: {
    toolType: "broaden" | "reframe" | "subvert";
    count: number;
  }[];

  // Mirror 결과
  mirror?: MirrorResult;
}

interface ConversationMessage {
  role: "user" | "assistant";
  content: string;
  timestamp: string;
}
```

### 3-5. MirrorResult (성찰 피드백)

```typescript
interface MirrorResult {
  // 오늘의 관찰 (1~2개)
  observations: {
    text: string;          // "오늘은 완성도보다 독특함을 택했네요."
    valueTags: ValueTag[];
    tone: "neutral" | "encouraging" | "curious";
  }[];

  // 누적 패턴과의 비교 (있으면)
  patternNote?: string;    // "지난주보다 다른 시각 도구를 더 많이 써봤어요."

  // 다음 미션 제안 이유
  nextSuggestion?: {
    reason: string;        // "관점 전환 미션을 더 해보면 좋겠어요"
    categoryHint: MissionCategory;
  };
}
```

### 3-6. UserProfile (사용자)

```typescript
interface UserProfile {
  id: string;
  name: string;             // "서연"
  age: number;              // 12
  createdAt: string;

  // 누적 통계
  stats: {
    totalMissions: number;
    currentStreak: number;   // 연속 참여 일수
    longestStreak: number;
    totalMinutes: number;
  };

  // 발견 패턴 (5가지 축)
  discoveries: {
    worldPreference: DiscoveryInsight;       // 어떤 세계에 끌리는지
    valueOrientation: DiscoveryInsight;      // 무엇을 중요하게 여기는지
    roleEnergy: DiscoveryInsight;            // 어떤 역할에 에너지가 생기는지
    decisionStyle: DiscoveryInsight;         // 어떤 방식으로 결정하는지
    tonePreference: DiscoveryInsight;        // 어떤 분위기를 선호하는지
  };

  // 관심 영역 맵
  interestMap: {
    category: string;       // "디자인 & 창작"
    score: number;          // 0~100
    trend: "up" | "stable" | "exploring";
  }[];
}

interface DiscoveryInsight {
  label: string;            // "어떤 세계에 끌리는지"
  summary: string;          // "우주보다 사람들이 사는 마을 이야기에 더 오래 머물러요"
  dataPoints: number;       // 이 인사이트의 근거가 된 선택 횟수
  confidence: "low" | "medium" | "high";  // 데이터 충분성
}
```

### 4-3. 미션 플레이 (`/mission/[id]/play`) — 핵심 화면

**목적**: 선택 → AI 대화 → 확장 도구 사용의 전체 체험

이 화면은 **3개의 페이즈**로 나뉩니다:

#### Phase A: 선택 (Choice)

```
┌─────────────────────────────────┐
│  화성 첫 도시의 시장     Step 1/3  │
│  ─────────────────────────────  │
│                                 │
│   농장과 병원 중 어디에            │

│                                 │
│   시장으로서 첫 번째               │
│   결정을 내려야 해.               │
│                                 │
│                                 │
│  ┌─────────────────────────┐    │
│  │  🌾 농장에 먼저 보낸다     │    │
│  │                          │    │
│  │  식량이 없으면 모두가       │    │
│  │  오래 버틸 수 없으니까     │    │
│  └─────────────────────────┘    │
│                                 │
│  ┌─────────────────────────┐    │
│  │  🏥 병원에 먼저 보낸다     │    │
│  │                          │    │
│  │  지금 아픈 사람이 있는데   │    │
│  │  나중 일을 먼저 생각할     │    │
│  │  수는 없으니까            │    │
│  └─────────────────────────┘    │
│                                 │
│  ┌─────────────────────────┐    │
│  │  ⚖️ 둘 다 반씩 나눈다     │    │
│  │                          │    │
```

**UX**:
- 선택지는 큰 카드로 (터치 친화적)
- 선택하면 짧은 확인 애니메이션 후 Phase B로
- 선택 이유를 짧게 적을 수 있는 선택적 입력란 ("왜 그렇게 골랐어?" — 스킵 가능)

```
┌─────────────────────────────────┐
│  화성 첫 도시의 시장     Step 2/3  │
│  ─────────────────────────────  │
│                                 │

│  │ 시장님, 병원을 선택하셨군요. │  │
│  │ 주민들한테 어떻게           │  │
│  │ 설명할 건가요?             │  │
│  │ 농장 쪽 사람들은           │  │
│  │ 불안해할 수도 있어요.       │  │
│  └───────────────────────────┘  │
│                                 │
│         ┌─ 나 ──────────────┐   │
│         │ 당장 아픈 사람이    │   │
│         │ 있으니까 병원이     │   │
│         │ 먼저라고 설명할게   │   │
│         └───────────────────┘   │
│                                 │
│  ┌─ AI ─────────────────────┐   │
│  │ 그 논리는 설득력 있어요.    │   │
│  │ 근데 농장 팀장이 이렇게     │   │
│  │ 물어보면 어떡하죠?         │   │
│  │                           │   │
│  │ "3주 뒤에 식량이 부족해지면 │   │
│  │  그때는 어떻게 할 건가요?"  │   │
│  └───────────────────────────┘  │
│                                 │
│  ┌───────────────────────────┐  │
│  │  메시지 입력...        전송  │  │
│  └───────────────────────────┘  │
│                                 │
│  ── 생각 도구 ──                  │
│  ┌────┐ ┌────┐ ┌────┐          │
│  │ 🔭 │ │ 🔄 │ │ 🌀 │          │
│  │넓히기│ │시각 │ │반전 │         │
│  └────┘ └────┘ └────┘          │
│                                 │
└─────────────────────────────────┘
```

**UX 디테일**:
- 채팅 인터페이스이지만 일반 챗봇과 다름
  - AI가 먼저 말을 걸고, 아이의 선택에 대해 질문함
  - 정보를 주는 게 아니라 **생각을 확장시키는 질문**을 함
  - 3~5회 주고받으면 자연스럽게 Phase C로 전환 유도
- **생각 도구** 3개 버튼이 하단에 항상 보임:
  - 🔭 **넓혀보기**: "이 결정의 다른 가능성을 보여줘"
  - 🔄 **다른 시각**: "다른 사람의 입장에서는 어떻게 보일까?"
  - 🌀 **이상하게**: "완전히 반대로 생각하면?"
- 생각 도구를 누르면 AI가 해당 관점에서 새로운 아이디어를 제시
- AI는 **절대 정답을 알려주지 않음** — 선택지를 열어줄 뿐

**AI 페르소나 규칙**:
- 미션의 세계관 안에서 역할을 가짐 (부시장, 동료 디자이너 등)
- 반말 사용 (친구처럼)
- 짧은 문장 (3줄 이하로 끊기)
- 아이가 막히면 힌트를 주되, 답을 주지 않음
- 감정을 인정해줌 ("그렇게 느끼는 거 자연스러운 거야")

#### Phase C: 마무리 유도

```
│  └───────────────────────────┘  │
```

- 강제 종료는 없음 — 아이가 더 대화하고 싶으면 계속 가능
- 다만 7분 경과 시 부드럽게 "슬슬 정리해볼까?" 제안

## 7. 개발 우선순위

더미 인터페이스 구현 순서:

| 순서 | 화면 | 의존성 | 확정되는 것 |
|------|------|--------|------------|
| 1 | `/mission/[id]` (인트로) | Mission 데이터 모델 | 미션 카드 구조 |
| 2 | `/mission/[id]/play` (체험) | Choice, ConversationMessage | AI 대화 인터페이스 + 확장 도구 |
| 3 | `/mission/[id]/mirror` (거울) | MirrorResult | 성찰 피드백 구조 |
| 4 | `/home` (홈) | Mission[], MissionSession[] | 미션 목록/상태 관리 |
| 5 | `/profile` (내 기록) | UserProfile, DiscoveryInsight | 누적 데이터 시각화 |
| 6 | `/parent` (부모 대시보드) | UserProfile + 주간 집계 | 부모용 리포트 |

## 8. 열린 질문 (논의 필요)

1. **미션 해금 방식**: 매일 1개 자동 배정? 아이가 고를 수 있게?
   - 추천: 매일 1개 자동 + 주말에 자유 선택 1개

2. **대화 길이 제한**: AI 대화를 몇 턴까지 허용?
   - 추천: 최소 3턴, 권장 5턴, 최대 10턴 (7분 타이머 soft limit)

3. **부모/아이 계정 관계**: 하나의 부모 계정에 아이 여러 명?
   - 추천: 부모 1 → 아이 N 구조

4. **온보딩**: 첫 미션 전에 프로필 생성? 바로 미션?
   - 추천: 이름 + 나이만 입력 → 바로 첫 미션 (마찰 최소화)

5. **오프라인/비로그인 체험**: 가입 전 체험판 제공?
   - 추천: 첫 1개 미션은 가입 없이 체험 가능

6. **생각 도구 사용 유도**: 아이가 안 쓰면 어떻게?
   - 추천: 3일차부터 AI가 자연스럽게 "혹시 다른 시각에서도 한번 볼래?" 제안

7. **콘텐츠 순서**: 카테고리 순환? 난이도 순?
   - 추천: 첫 7일은 5개 카테고리 각 1개씩 + 종합 2개 (고정 순서)
   - 8일차부터는 아이의 패턴에 맞춘 적응형 배정
