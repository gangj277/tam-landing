# Landing Page Specification

## Overview

이 문서는 부모를 대상으로 한 랜딩페이지의 전체 구조, 각 섹션의 구체적 콘텐츠, 감정 설계, 디자인 방향을 정의합니다.

### 핵심 설계 원칙

1. **부모의 감정 흐름을 설계한다** — 각 섹션은 이전 섹션의 감정 위에 쌓인다
2. **기능이 아니라 가치를 보여준다** — "이런 기능이 있습니다"가 아니라 "아이에게 이런 일이 생깁니다"
3. **설명하지 않고 체감시킨다** — 가능한 한 예시, 시나리오, 시각적 증거로 전달한다
4. **신뢰는 디자인 품질 자체에서 온다** — 프리미엄 브랜드 수준의 타이포그래피, 여백, 구성

### 전체 감정 여정

```
위기감 → 공감 → 리프레이밍 → 기대감 → 납득 → 감동 → 신뢰 → 행동
```

### 디자인 시스템

- **배경**: 밝은 크림 화이트 (#FAFAF8) — 따뜻하고 신뢰감 있는 톤
- **주 텍스트**: 딥 네이비 (#1A1A2E) — 권위와 안정감
- **악센트 1**: 웜 코랄 (#E8614D) — 에너지, 핵심 강조
- **악센트 2**: 소프트 인디고 (#4A5FC1) — 지적 신뢰감
- **보조**: 웜 골드 (#D4A853) — 프리미엄 느낌의 포인트
- **타이포**: 한글 본문 Pretendard, 영문 헤드라인 보조 DM Serif Display
- **전체 톤**: Apple Education + 좋은 사립학교 브로셔 + 프리미엄 매거진

## Section 1: Hero

### 감정 목표
> "AI가 세상을 바꾸고 있다는 건 알고 있다. 근데 우리 아이는...?"

부모가 이미 느끼고 있는 막연한 불안을 정확히 건드린다. 새로운 정보를 주는 게 아니라, 이미 있는 감정을 수면 위로 끌어올린다.

### 레이아웃

전체 뷰포트를 차지하는 풀스크린 히어로. 상단에 네비게이션, 중앙에 메인 카피, 하단에 CTA.

```
┌─────────────────────────────────────────────────┐
│  [로고]                          [대기자 신청]   │
│                                                  │
│                                                  │
│         세상은 AI로 빠르게 바뀌고 있습니다.         │
│                                                  │
│     우리 아이는 그 세상을 경험하고 있나요?          │
│                                                  │
│         (서브카피 1~2줄)                          │
│                                                  │
│          [ 얼리 액세스 신청하기 ]                  │
│                                                  │
│              ↓ 스크롤                             │
└─────────────────────────────────────────────────┘
```

### 카피

**메인 헤드라인:**
```
세상은 AI로 빠르게 바뀌고 있습니다.
우리 아이는 그 세상을 경험하고 있나요?
```

**서브카피:**
```
매일 5분, AI와 함께 새로운 세계를 탐험하며
생각하는 힘과 자기만의 기준을 키워가는 경험 플랫폼
```

### 디자인 디테일

- 배경: 크림 화이트 베이스에 매우 은은한 그라데이션 (상단 약간 쿨톤 → 하단 웜톤)
- 헤드라인: 딥 네이비, 굵은 웨이트(Bold/Black), 충분히 크게 (모바일 32px, 데스크탑 56px)
- "AI로 빠르게 바뀌고 있습니다"에서 "AI"만 코랄 악센트로 강조
- 서브카피: 본문보다 약간 밝은 그레이(#4A4A5A), Regular 웨이트
- CTA 버튼: 코랄 배경 + 화이트 텍스트, 라운드 코너, 호버 시 살짝 스케일업
- 하단 스크롤 유도: 작은 아래 화살표 + 부드러운 바운스 애니메이션

### 배경 비주얼 (선택지)

**Option A — 추상 일러스트레이션:**
화면 양 옆에 반투명하게 떠 있는 다양한 세계의 아이콘들 (우주선, 도시, 바다, 실험실, 팔레트 등). 중앙 텍스트를 방해하지 않으면서 "다양한 세계"를 암시.

**Option B — 미니멀 모션:**
배경 없이 깔끔한 화이트. 대신 헤드라인의 "AI"가 타이핑되듯 나타나고, 아래 서브카피가 페이드인. 텍스트의 힘만으로 승부.

**추천: Option B** — 첫 인상에서 "이 브랜드는 격이 다르다"는 느낌을 줘야 하므로 비주얼 노이즈를 최소화.

## Section 2: 위기감 — "AI 시대, 아이에게 진짜 필요한 건"

### 감정 목표
> "맞아, 나도 이런 생각 계속 했는데... 역시 뭔가 해야 하는 거 맞지?"

부모가 이미 느끼는 불안을 데이터와 사례로 구체화한다. 단, 공포 마케팅이 아니라 "현실 인식"의 톤으로.

### 레이아웃

```
┌─────────────────────────────────────────────────┐
│                                                  │
│            숫자 3개 (가로 배치)                    │
│     ┌──────┐   ┌──────┐   ┌──────┐              │
│     │ 통계1 │   │ 통계2 │   │ 통계3 │             │
│     └──────┘   └──────┘   └──────┘              │
│                                                  │
│              본문 카피 (2~3단락)                   │
│                                                  │
└─────────────────────────────────────────────────┘
```

### 콘텐츠

**섹션 타이틀 (작게, 라벨 스타일):**
```
AI 시대의 현실
```

**통계 카드 3개:**

| 수치 | 설명 | 출처 |
|------|------|------|
| 65% | 지금의 초등학생이 성인이 되면, 현재 존재하지 않는 직업을 갖게 됩니다 | World Economic Forum |
| 2배 | AI를 활용하는 사람과 그렇지 않은 사람의 업무 생산성 차이 | McKinsey, 2024 |
| 5분 | 하루 5분의 다양한 경험이 아이의 사고 유연성을 키운다 | 스탠포드 교육 연구 |

> 참고: 실제 출시 전 모든 통계는 검증 필요. 위는 방향성 예시.

**본문 카피:**
```
AI는 더 이상 먼 미래의 이야기가 아닙니다.
이미 우리의 일상, 일, 교육, 창작의 방식을 근본적으로 바꾸고 있습니다.

그런데 우리 아이들의 일상은 어떤가요?
학원, 숙제, 유튜브, 게임 —
아이들이 경험하는 세계는 여전히 너무 좁습니다.

AI 시대에 정말 필요한 건 코딩 능력이 아닙니다.
다양한 세계를 만나고, 스스로 생각하고, 자기만의 기준으로 판단하는 힘입니다.
```

### 디자인 디테일

- 통계 카드: 큰 숫자는 코랄 악센트 + DM Serif (또는 Pretendard Bold), 설명은 본문 서체
- 카드 배경: 약간 밝은 톤의 카드 (#FFFFFF) + 미세한 그림자
- 스크롤 시 숫자가 카운트업 애니메이션으로 등장 (0에서 65로 올라가는 등)
- 본문의 "코딩 능력이 아닙니다" 부분에 취소선 또는 흐릿한 처리로 리프레이밍 효과
- 여백 넉넉하게 — 읽기 편안해야 함

### 필요 에셋

- 통계 수치 검증 후 최종 데이터 확정
- 카운트업 인터랙션 (framer-motion 또는 CSS)

## Section 3: 리프레이밍 — "경험이 생각을 만듭니다"

### 감정 목표
> "아, 코딩이나 프롬프트를 가르치는 게 아니라, 다양한 경험 자체가 중요한 거구나"

부모의 기존 프레임("AI 교육 = 코딩, 프롬프트")을 전환시킨다. 이 섹션이 설득의 핵심 분기점.

### 레이아웃

```
┌─────────────────────────────────────────────────┐
│                                                  │
│            큰 인용문 스타일 카피                    │
│                                                  │
│  ─────────────────────────────────────────────── │
│                                                  │
│     (왼쪽)              (오른쪽)                  │
│     지금 아이들의        우리가 만드는              │
│     디지털 경험          새로운 경험                │
│     ┌─────────┐        ┌─────────┐              │
│     │ 수동적   │   →    │ 능동적   │              │
│     │ 반복적   │   →    │ 탐험적   │              │
│     │ 소비     │   →    │ 선택과   │              │
│     │          │        │ 판단     │              │
│     └─────────┘        └─────────┘              │
│                                                  │
└─────────────────────────────────────────────────┘
```

### 콘텐츠

**메인 카피 (인용문 스타일, 크게):**
```
"아이들은 자기 안만 들여다본다고
 스스로를 이해하게 되지 않습니다.

 세상의 다양한 결을 만나고,
 그 앞에서 자기 방식으로 반응할 때
 비로소 자기 자신을 이해하게 됩니다."
```

**비교 테이블:**

| 지금 아이들의 디지털 경험 | 우리가 여는 새로운 경험 |
|---|---|
| 정해진 콘텐츠를 소비한다 | 매일 새로운 세계에 직접 들어간다 |
| AI가 정답을 알려준다 | AI로 가능성을 넓히고, 스스로 고른다 |
| 결과가 항상 같다 | 내 선택에 따라 결과가 달라진다 |
| 끝나면 남는 게 없다 | 내 선택의 패턴이 쌓여간다 |

### 디자인 디테일

- 인용문: 큰 따옴표(") 장식 + 서체 약간 이탤릭 느낌, 딥 네이비
- 비교 테이블: 왼쪽은 연한 그레이 톤 + 약간 바랜 느낌, 오른쪽은 화이트 + 코랄 악센트 하이라이트
- 화살표(→) 대신 부드러운 곡선 연결선 사용 가능
- 스크롤 진입 시 왼쪽이 먼저 보이고, 오른쪽이 슬라이드인 또는 페이드인

### 필요 에셋

- 비교 테이블 인터랙션 (스크롤 트리거)
- 인용 부호 장식 SVG

## Section 4: 제품 소개 — "매일 5분, 새로운 세계가 열립니다"

### 감정 목표
> "오, 이런 게 있구나. 구체적으로 어떻게 되는 거지?"

리프레이밍 후 자연스럽게 제품을 소개한다. 아직 기능 나열이 아닌, "아이에게 일어나는 일"로 설명.

### 레이아웃

```
┌─────────────────────────────────────────────────┐
│                                                  │
│         섹션 타이틀 + 서브카피                     │
│                                                  │
│  ┌───────────────────────────────────────────┐   │
│  │                                           │   │
│  │         앱 화면 목업 / 인터랙티브 데모       │   │
│  │                                           │   │
│  │   "오늘의 미션"                            │   │
│  │                                           │   │
│  │   넌 화성 첫 도시의 시장이야.               │   │
│  │   물이 부족해 —                            │   │
│  │   농장과 병원 중 어디에 먼저 줄까?           │   │
│  │                                           │   │
│  │    [🌾 농장]    [🏥 병원]                  │   │
│  │                                           │   │
│  └───────────────────────────────────────────┘   │
│                                                  │
│       아래에 3개의 핵심 가치 카드                   │
│                                                  │
└─────────────────────────────────────────────────┘
```

### 콘텐츠

**섹션 타이틀:**
```
매일 5분, 새로운 세계가 열립니다
```

**서브카피:**
```
앱을 열면 오늘의 세계가 바로 시작됩니다.
긴 설명도, 복잡한 설정도 없습니다.
아이는 새로운 역할이 되어 짧고 강렬한 경험을 합니다.
```

**인터랙티브 데모:**

실제 앱 경험을 작은 규모로 재현한 미니 인터랙션. 부모가 직접 선택을 눌러볼 수 있다.

시나리오:
```
[미션 카드]
"넌 화성 첫 도시의 시장이야.
 도시를 세운 지 3개월, 물 공급이 부족해지고 있어.
 농장과 병원 중 한 곳에만 먼저 물을 보낼 수 있어."

[선택지 A] 농장 — 식량이 없으면 모두가 위험해
[선택지 B] 병원 — 아픈 사람부터 살려야지

→ 선택 시:
"당신은 [농장/병원]을 선택했습니다.
 아이도 이런 선택을 매일 합니다.
 그리고 그 선택들이 쌓여, 아이만의 생각지도가 만들어집니다."
```

**3개 핵심 가치 카드:**

```
┌─────────────────┐  ┌─────────────────┐  ┌─────────────────┐
│  🌍              │  │  🧠              │  │  🤖              │
│  경험의 지평 확장  │  │  자기이해의 성장  │  │  AI 활용 감각     │
│                  │  │                  │  │                  │
│  매일 새로운      │  │  반복된 선택의    │  │  AI를 두려워하지  │
│  세계와 역할을    │  │  패턴에서 아이만의 │  │  않고, 맹신하지   │
│  만납니다        │  │  가치관이         │  │  않고, 직접 다루는 │
│                  │  │  드러납니다       │  │  감각을 키웁니다   │
└─────────────────┘  └─────────────────┘  └─────────────────┘
```

### 디자인 디테일

- 데모 영역: 앱 프레임 목업 안에 실제 인터랙션. 밝은 배경 카드 + 그림자
- 선택 버튼: 충분히 크고, 호버 시 살짝 올라오는 느낌
- 선택 후 트랜지션: 부드럽게 결과 텍스트로 전환
- 가치 카드: 동일한 크기, 아이콘 상단, 텍스트 하단. 호버 없이 깔끔하게
- 이 섹션 배경: 아주 약간 따뜻한 톤 변화 (#F5F3EE 정도)

### 필요 에셋

- 앱 프레임 목업 (모바일 디바이스 프레임 또는 플로팅 카드)
- 인터랙티브 선택 컴포넌트 (클릭 → 결과 전환)
- 아이콘 3개 (경험, 자기이해, AI — 라인 아이콘 스타일)

## Section 5: 작동 방식 — "Encounter → Choice → Act → Mirror"

### 감정 목표
> "아, 이 앱이 정확히 어떤 구조로 돌아가는지 이해했다. 체계적이네."

부모가 제품의 작동 원리를 명확히 이해하고, "잘 설계된 제품"이라는 인식을 갖게 한다.

### 레이아웃

```
┌─────────────────────────────────────────────────┐
│                                                  │
│              섹션 타이틀                           │
│                                                  │
│   ① 세계 진입 → ② 가치 선택 → ③ 행동 → ④ 발견   │
│                                                  │
│    각 단계별 상세 설명 (스크롤 또는 탭)             │
│                                                  │
└─────────────────────────────────────────────────┘
```

### 콘텐츠

**섹션 타이틀:**
```
하루 한 번, 네 단계의 경험
```

**서브카피:**
```
매일의 경험은 같은 구조로 반복됩니다.
그래서 아이는 익숙한 리듬 속에서 매일 다른 세계를 경험합니다.
```

**4단계 상세:**

**Step 1 — 세계에 들어간다 (Encounter)**
```
오늘은 어떤 세계일까?
화성 도시의 시장, 해양 연구선의 선장, 미래 박물관의 큐레이터...
매일 전혀 다른 세계와 역할이 열립니다.
아이는 아직 만나보지 못한 상황 앞에 서게 됩니다.
```

**Step 2 — 선택한다 (Choice)**
```
정답은 없습니다. 가치의 선택만 있습니다.
효율과 공정 사이에서, 안전과 모험 사이에서,
나를 위한 것과 모두를 위한 것 사이에서
아이는 자기만의 기준으로 고릅니다.
```

**Step 3 — 행동한다 (Act)**
```
선택만 하고 끝나지 않습니다.
AI와 함께 아이디어를 넓히고, 다른 시각으로 바꿔보고,
직접 결과물을 만들어봅니다.
AI를 도구로 다루는 감각이 여기서 자랍니다.
```

**Step 4 — 발견한다 (Mirror)**
```
"오늘은 완성도보다 독특함을 택했네요."
"다른 사람의 반응을 많이 생각하면서 골랐어요."

평가가 아니라 거울입니다.
아이의 선택 패턴을 비춰주고,
스스로 자기 자신을 이해해가게 합니다.
```

### 디자인 디테일

- 4단계를 가로 타임라인으로 배치 (데스크탑) / 세로 스텝으로 (모바일)
- 각 단계에 번호 원형 뱃지(①②③④) — 인디고 악센트
- 현재 보고 있는 단계 강조, 나머지 살짝 투명
- 각 단계 전환 시 부드러운 슬라이드/페이드
- Step 4(Mirror)에 도달하면 약간의 글로우 효과로 "아하" 모먼트 강조

### 필요 에셋

- 각 단계별 일러스트 또는 아이콘 (선형, 미니멀)
- 타임라인 연결 라인 SVG
- 스크롤 기반 스텝 진행 인터랙션

## Section 7: 자기발견 — "아이의 선택이 말해주는 것들"

### 감정 목표
> "이건 다른 앱에 없는 거네. 아이가 이걸 하면서 자기 자신을 알아가게 되는 거구나..."

부모가 이 제품만의 핵심 차별점(Mirror/자기이해)에 감동하는 섹션. 감정적 클라이맥스.

### 레이아웃

```
┌─────────────────────────────────────────────────┐
│                                                  │
│            섹션 타이틀 + 서브카피                  │
│                                                  │
│  ┌────────────────────────────────────────────┐  │
│  │                                            │  │
│  │          Weekly Mirror 목업                 │  │
│  │                                            │  │
│  │  "이번 주 ○○이의 경험 지도"                 │  │
│  │                                            │  │
│  │  - 5번의 경험 중 4번 '독특함'을 선택했어요    │  │
│  │  - 다른 사람을 돕는 역할에서 에너지가 높았어요 │  │
│  │  - 빠르게 결정하는 편이지만,                  │  │
│  │    이번 주엔 여러 안을 비교하기 시작했어요     │  │
│  │                                            │  │
│  │         [관심 영역 비주얼 맵]                │  │
│  │                                            │  │
│  └────────────────────────────────────────────┘  │
│                                                  │
│         아래에 부모 코멘트 스타일 인용문             │
│                                                  │
└─────────────────────────────────────────────────┘
```

### 콘텐츠

**섹션 타이틀:**
```
놀고 있을 뿐인데, 아이가 보입니다
```

**서브카피:**
```
매일의 선택은 조용히 쌓입니다.
일주일이 지나면, 아이가 어떤 세계에 끌리는지,
무엇을 중요하게 여기는지, 어떻게 생각하는지가
부드럽게 드러납니다.

이건 테스트 결과가 아닙니다.
아이의 진짜 반응에서 나온 이야기입니다.
```

**Weekly Mirror 예시 콘텐츠:**

```
이번 주 서연이의 경험 지도

🌍 탐험한 세계: 화성 도시, 동물구조센터, 미래학교, 해양연구선, 마을 중재

💡 발견된 패턴들

  "완성도보다 독특함을 자주 선택했어요"
  — 5번의 경험 중 4번, 정돈된 결과보다 새로운 시도를 골랐습니다.

  "다른 사람을 돕는 역할에서 몰입도가 높았어요"
  — 시장, 중재자, 디자이너 역할에서 특히 오래 머물렀습니다.

  "이번 주엔 여러 안을 비교하기 시작했어요"
  — 처음엔 바로 골랐는데, 후반부터 AI로 다른 버전을 만들어보기 시작했어요.

🗺️ 관심 영역 맵
  [디자인·창작 ████████░░]  높음
  [리더십·관리 ███████░░░]  높음
  [과학·탐구   ████░░░░░░]  보통
  [감정·관계   ███░░░░░░░]  아직 탐색 중
```

**부모 반응 인용문 (향후 실제 테스터 피드백으로 교체):**
```
"아이가 이런 걸 중요하게 여기는지 처음 알았어요.
 시험 점수로는 절대 볼 수 없는 것들이더라고요."
```

### 디자인 디테일

- Mirror 목업: 앱 화면 프레임 안에 넣어서 실제 앱 느낌
- 관심 영역 맵: 가로 바 차트 or 레이더 차트 (간결하고 예쁘게)
- 패턴 설명: 각각 작은 아이콘 + 짧은 문장. 평가가 아닌 관찰 톤
- 인용문: 큰 따옴표 + 이탤릭, 하단에 작게 "— 12세 자녀를 둔 어머니"
- 전체 섹션 배경: 아주 약간 따뜻한 톤 (#F8F5F0) — 감정적 구간이므로 부드럽게

### 필요 에셋

- Weekly Mirror UI 목업 (앱 프레임 안)
- 관심 영역 차트 컴포넌트 (바 차트 또는 레이더)
- 인용문 장식 요소

## Section 8: AI 리터러시 — "AI를 다루는 감각이 자랍니다"

### 감정 목표
> "단순히 AI를 쓰는 게 아니라, AI를 제대로 다루는 법을 배우는 거구나"

부모에게 "AI 리터러시"가 구체적으로 뭔지 보여준다. 프롬프트 교육이 아닌, 도구적 감각.

### 레이아웃

```
┌─────────────────────────────────────────────────┐
│                                                  │
│           섹션 타이틀 + 서브카피                   │
│                                                  │
│    ┌─────────────────────────────────────────┐   │
│    │                                         │   │
│    │     AI 도구 버튼 인터랙션 데모            │   │
│    │                                         │   │
│    │  아이의 첫 번째 아이디어:                 │   │
│    │  "동물구조센터 포스터 — 강아지 사진"       │   │
│    │                                         │   │
│    │  [더 넓혀보기] [다른시각] [이상하게바꾸기]  │   │
│    │                                         │   │
│    │  → 버튼 클릭 시 변화된 결과 표시          │   │
│    │                                         │   │
│    └─────────────────────────────────────────┘   │
│                                                  │
│         핵심 메시지 3줄                           │
│                                                  │
└─────────────────────────────────────────────────┘
```

### 콘텐츠

**섹션 타이틀:**
```
AI를 도구로 다루는 감각
```

**서브카피:**
```
아이는 AI에게 질문하지 않습니다.
AI를 이용해 결과를 바꿉니다.
```

**AI 도구 인터랙션 데모:**

아이가 동물구조센터 포스터를 만드는 상황. AI 도구 버튼을 누르면 결과가 변한다.

```
[원래 아이디어]
"귀여운 강아지 사진이 있는 동물구조센터 포스터"

[더 넓혀보기] 클릭 →
  "강아지뿐 아니라, 구조된 동물들의 '입양 전/후' 사진을 나란히 놓는 건 어때?"
  "포스터 대신, 사람들이 직접 이름을 지어주는 참여형 캠페인은?"
  "동물이 직접 말하는 것처럼 1인칭 시점의 포스터는?"

[다른 시각으로 보기] 클릭 →
  "만약 동물의 입장에서 본다면?"
  "만약 5살 아이에게 보여줄 포스터라면?"
  "만약 동물을 싫어하는 사람도 멈추게 하려면?"

[이상하게 바꾸기] 클릭 →
  "동물이 사람을 구조하는 반전 포스터?"
  "100년 뒤, 동물이 멸종 위기인 세상의 포스터?"
```

**핵심 메시지:**
```
AI가 만들어준 것을 그대로 쓰는 게 아닙니다.
여러 가능성을 보고, 비교하고, 고르고, 수정합니다.

이것이 AI 시대에 진짜 필요한 리터러시입니다.
프롬프트 문법이 아니라, 도구를 다루는 감각.
```

### 디자인 디테일

- 데모 영역: 카드 UI 안에 원래 아이디어 + AI 도구 버튼들
- 버튼: 인디고 아웃라인 스타일, 클릭 시 필 상태로 전환
- 결과: 버튼 아래에 슬라이드다운으로 AI 제안 3개 나타남
- 부모가 직접 버튼을 눌러보면서 "아 이런 식으로 AI를 쓰는 거구나" 체감
- 본문의 "프롬프트 문법이 아니라"에서 "프롬프트 문법"은 취소선, "도구를 다루는 감각"은 코랄 강조

### 필요 에셋

- AI 도구 버튼 인터랙티브 컴포넌트
- 3개 도구별 AI 응답 콘텐츠 (하드코딩)
- 전환 애니메이션

## Section 9: 첫 7일 — "첫 일주일의 여정"

### 감정 목표
> "온보딩도 이렇게 세심하게 설계했구나. 진짜 전문적이다."

첫 7일이 랜덤이 아니라 의도적으로 설계된 '샘플링 주간'임을 보여주어 제품의 깊이를 인식시킨다.

### 레이아웃

```
┌─────────────────────────────────────────────────┐
│                                                  │
│             섹션 타이틀 + 서브카피                 │
│                                                  │
│  Day 1  Day 2  Day 3  Day 4  Day 5  Day 6  Day 7│
│  ──●──────●──────●──────●──────●──────●──────●── │
│                                                  │
│  [선택된 Day의 상세 설명 카드]                     │
│                                                  │
└─────────────────────────────────────────────────┘
```

### 콘텐츠

**섹션 타이틀:**
```
첫 7일, 아이만의 지도가 시작됩니다
```

**서브카피:**
```
첫 일주일은 특별합니다.
다양한 결의 경험을 하나씩 맛보면서,
아이의 초기 반응 지도가 만들어지는 시간입니다.
```

**7일 타임라인:**

| Day | 이름 | 한 줄 설명 | 목적 (부모용, 작게) |
|-----|------|-----------|-------------------|
| 1 | 낯선 세계 | 처음 보는 세계에 뛰어들어 봐 | 호기심 반응 관찰 |
| 2 | 취향 선택 | 같은 내용, 다른 분위기 — 어떤 게 끌려? | 미적 감각 탐색 |
| 3 | 역할 실험 | 같은 문제, 두 개의 역할 — 어느 쪽이 나야? | 역할 선호 파악 |
| 4 | 가치 딜레마 | 둘 다 맞는데, 하나만 골라야 한다면? | 가치관 관찰 |
| 5 | 탐색 스타일 | 바로 정할래, 여러 개 보고 고를래? | 의사결정 방식 |
| 6 | 현실 연결 | 오늘은 앱 밖 세상을 한번 보자 | 관찰력 확인 |
| 7 | 첫 번째 거울 | 이번 주 네가 고른 것들을 돌아보자 | 자기인식의 시작 |

### 디자인 디테일

- 가로 타임라인: 7개 점(●)이 선으로 연결. 각 점에 Day 번호
- 클릭/호버 시 해당 Day의 상세 카드가 아래에 표시
- Day 7에 작은 별 또는 글로우 효과 — "첫 번째 거울" 강조
- 전체적으로 부드러운 인디고 톤 타임라인

### 필요 에셋

- 타임라인 컴포넌트 (가로 도트 + 선)
- Day별 상세 카드 전환 인터랙션
- Day 7 특별 비주얼 (글로우/별)

## Section 10: 신뢰 — "안심하고 맡길 수 있습니다"

### 감정 목표
> "AI라서 걱정했는데, 이 정도면 믿을 수 있겠다."

부모의 마지막 허들(AI 안전성, 스크린타임, 데이터)을 제거한다.

### 레이아웃

```
┌─────────────────────────────────────────────────┐
│                                                  │
│             섹션 타이틀                           │
│                                                  │
│   ┌──────┐    ┌──────┐    ┌──────┐    ┌──────┐  │
│   │안전한 │    │짧은   │    │평가   │    │부모   │  │
│   │AI     │    │사용   │    │없음   │    │대시보드│  │
│   │       │    │시간   │    │       │    │       │  │
│   └──────┘    └──────┘    └──────┘    └──────┘  │
│                                                  │
└─────────────────────────────────────────────────┘
```

### 콘텐츠

**섹션 타이틀:**
```
부모님이 안심할 수 있는 이유
```

**4개 신뢰 카드:**

**1. 안전한 AI**
```
아이에게 부적절한 콘텐츠가 생성되지 않도록
다층 안전 필터를 적용합니다.
AI는 정답을 주지 않고,
가능성을 열어주는 도구로만 작동합니다.
```

**2. 하루 5~10분**
```
한 번의 경험은 5분에서 10분이면 끝납니다.
끝없이 빠져드는 구조가 아닙니다.
하루 한 번, 짧고 강하게.
```

**3. 평가하지 않습니다**
```
점수도, 등급도, 순위도 없습니다.
아이의 선택을 평가하지 않고,
패턴을 비춰줄 뿐입니다.
"넌 이런 아이야"가 아니라
"요즘은 이런 쪽을 많이 고르고 있어요."
```

**4. 부모를 위한 요약**
```
매주 아이의 경험 요약을 받아보실 수 있습니다.
아이가 어떤 세계에 끌렸는지,
어떤 선택을 했는지,
시험 점수로는 보이지 않는 것들이 보입니다.
```

### 디자인 디테일

- 4개 카드: 2x2 그리드 (데스크탑) / 1열 스택 (모바일)
- 각 카드 상단에 라인 아이콘 (방패, 시계, 하트, 차트)
- 배경: 가장 밝은 톤 (#FFFFFF) — 깨끗하고 안심
- 카드 보더: 연한 그레이, 미세한 그림자
- 전체적으로 차분하고 안정적인 느낌

### 필요 에셋

- 4개 아이콘 (라인 스타일, 1색)
- 카드 컴포넌트

## Section 11: CTA — "시작하기"

### 감정 목표
> "이건 해봐야겠다. 빨리 써보고 싶다."

전체 내러티브의 결론. 행동으로 전환.

### 레이아웃

```
┌─────────────────────────────────────────────────┐
│                                                  │
│     (배경: 약간 따뜻한 톤, 또는 그라데이션)        │
│                                                  │
│     AI 시대, 아이에게 줄 수 있는                   │
│     가장 좋은 경험의 시작                          │
│                                                  │
│     ┌─────────────────────────────────────┐      │
│     │  이메일 입력                [신청하기] │      │
│     └─────────────────────────────────────┘      │
│                                                  │
│     "현재 000명의 부모님이 대기 중입니다"           │
│                                                  │
│     작은 보충 텍스트:                              │
│     무료 체험 · 언제든 취소 가능 · 개인정보 보호     │
│                                                  │
└─────────────────────────────────────────────────┘
```

### 콘텐츠

**메인 카피:**
```
AI 시대,
아이에게 줄 수 있는 가장 좋은 경험의 시작
```

**서브카피:**
```
먼저 경험해보실 분들을 모집하고 있습니다.
얼리 액세스에 등록하시면,
출시 즉시 무료로 초대해 드립니다.
```

**소셜 프루프:**
```
현재 {count}명의 부모님이 대기 중입니다
```

**안심 텍스트:**
```
무료 체험 · 언제든 취소 가능 · 스팸 없음
```

### 디자인 디테일

- 배경: 아주 부드러운 그라데이션 (크림 → 살짝 핑크빛)
- 메인 카피: 가장 큰 사이즈는 아니지만, 웨이트가 강하고 중심에 위치
- 이메일 인풋: 큰 사이즈, 라운드 코너, 우측에 CTA 버튼 인라인
- CTA 버튼: 코랄 배경 + 화이트 텍스트, "얼리 액세스 신청"
- 대기자 수: 작은 텍스트, 숫자만 코랄으로 강조
- 안심 텍스트: 가장 작은 사이즈, 그레이

### 필요 에셋

- 이메일 수집 폼 컴포넌트
- 대기자 카운터 (초기에는 정적 숫자, 추후 실제 연동)
